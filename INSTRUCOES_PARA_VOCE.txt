🎯 COMO FAZER O DEPLOY? (SUPER FÁCIL!)
════════════════════════════════════════════════════════════════

PASSO 1: Leia o resumo
   cat DEPLOY_READY.md

PASSO 2: Prepare o script (só uma vez)
   cd DocCollab
   chmod +x deploy.sh

PASSO 3: Execute o deploy
   ./deploy.sh SEU_IP root

   Exemplo real:
   ./deploy.sh 142.93.123.45 root

PASSO 4: Aguarde ~5 minutos
   O script fará TUDO automaticamente:
   - Instalar dependências
   - Configurar servidor
   - Configurar Nginx, Gunicorn, Supervisor
   - Configurar firewall
   - Iniciar aplicação

PASSO 5: Acesse no navegador
   http://SEU_IP

PASSO 6: Faça login
   Email: admin@doccollab.com
   Senha: admin123
   
   ⚠️ IMPORTANTE: Altere a senha imediatamente!

════════════════════════════════════════════════════════════════
🔒 CONFIGURAR SSL (OPCIONAL MAS RECOMENDADO)
════════════════════════════════════════════════════════════════

Se você tem um domínio (ex: doccollab.com):

1. Conecte ao servidor:
   ssh root@SEU_IP

2. Execute o Certbot:
   certbot --nginx -d seu-dominio.com

3. Pronto! Seu site estará em:
   https://seu-dominio.com 🔐

📊 COMANDOS ÚTEIS APÓS O DEPLOY
════════════════════════════════════════════════════════════════

Ver logs em tempo real:
   ssh root@SEU_IP 'tail -f /var/log/doccollab/error.log'

Reiniciar aplicação:
   ssh root@SEU_IP 'supervisorctl restart doccollab'

Ver status:
   ssh root@SEU_IP 'supervisorctl status doccollab'

Atualizar aplicação (quando fizer mudanças):
   chmod +x update.sh
   ./update.sh SEU_IP root

🆘 PROBLEMAS COMUNS E SOLUÇÕES
════════════════════════════════════════════════════════════════

Erro 502 Bad Gateway:
   ssh root@SEU_IP 'supervisorctl restart doccollab'

Aplicação não inicia:
   ssh root@SEU_IP 'tail -n 50 /var/log/doccollab/error.log'

Permissões negadas:
   ssh root@SEU_IP 'chown -R doccollab:doccollab /var/www/doccollab'

Socket.IO não funciona:
   ssh root@SEU_IP 'nginx -t && systemctl reload nginx'

✨ HABILITAR IA REAL (OPCIONAL)
════════════════════════════════════════════════════════════════

Se quiser usar IA real (OpenAI, etc.):

1. Obter API Key da OpenAI
2. Conectar ao servidor: ssh root@SEU_IP
3. Editar configuração: nano /var/www/doccollab/.env
4. Adicionar:
   LLM_MODE=llm
   LLM_API_KEY=sua-chave-aqui
   LLM_MODEL=gpt-3.5-turbo
5. Reiniciar: supervisorctl restart doccollab
